# Sentinel — Real-Mode Implementation Plan

Baby-step plan to move Sentinel from dummy mode to a real, onchain + persistent system. Each step is independently verifiable and can ship on its own. Order matches the agreed sequence (contracts first, persistence second).

## Phase 1 — Contracts (local only)

Goal: both Vyper contracts compile, tests are green, and a Python script can deploy them locally and exercise every method.

- **1.1** Install Ape + `ape-vyper` in `contracts/.venv`.
- **1.2** `ape compile` succeeds for `ReputationLedger.vy` and `OperatorPool.vy`.
- **1.3** Add `contracts/MockERC20.vy` and fix the `usdc` fixture so `test_operator_pool.py` no longer skips.
- **1.4** `ape test contracts/tests` is fully green.
- **1.5** Local deploy script: bring up an ape local network, deploy both contracts, call `write_batch` / `get_score` and `deposit` / `debit` / `balance` end-to-end from Python.

## Phase 2 — Persistence

- **2.1** `docker-compose up -d postgres` + `alembic upgrade head`; `ReputationStore` persists real rows; pytest hits a real DB.
- **2.2** `docker-compose up -d redis`; `ReputationCache` reads/writes with TTL; cadence signal works across two uvicorn workers.

## Phase 3 — Backend ↔ chain

- **3.1** `ReputationLedger` adapter reads from the deployed contract via web3.py. Config flag to fall back to dummy.
- **3.2** Batched writer worker: drains pending signals, posts `write_batch` on an interval. Kill switch for fallback.

## Phase 4 — Payments

- **4.1** Circle sandbox + fiat → USDC top-up to `OperatorPool`.
- **4.2** x402 per-call debit in `payments/circle.py`; batched settlement calls `OperatorPool.debit`.
- **4.3** End-to-end: card → pool credits → evals debit → balance drops in dashboard.

## Phase 5 — Testnet

- **5.1** Deploy both contracts to Arc testnet.
- **5.2** Run the full flow (Phase 4 + backend + dashboard) against testnet + Circle sandbox.
